# STM32F407 智能手表移植 - FreeRTOS 重新移植总结

## 会话背景

之前完成的 F103→F407 移植中，FreeRTOS 使用的是 Cortex-M3 版 port.c，导致 HardFault 崩溃。本次会话彻底删除旧 FreeRTOS，从 GitHub 下载官方内核重新移植。

---

## 一、问题根因

串口调试显示所有外设初始化成功，但 `vTaskStartScheduler()` 后系统崩溃：
- CFSR = 0x00008200（LSPERR - Lazy State Preservation Error）
- pxDelayedTaskList 被破坏为 0x10F0F64C（非法地址）
- 根因：Cortex-M3 版 port.c 不保存 FPU 寄存器 S16-S31

---

## 二、FreeRTOS 重新移植

### 1. 删除旧文件，下载官方内核
- 删除整个 `FreeRTOS/` 目录
- `git clone https://github.com/FreeRTOS/FreeRTOS-Kernel.git`
- 复制 RVDS/ARM_CM4F 端口文件

### 2. 文件结构
```
FreeRTOS/
├── FreeRTOSConfig.h      # 精简配置
├── inc/                   # 内核头文件 (21个)
├── port/
│   ├── port.c            # 官方 RVDS/ARM_CM4F（含 FPU 支持）
│   ├── portmacro.h       # 端口宏定义
│   └── heap_4.c          # 动态内存分配
└── src/                   # 内核源码 (7个)
    ├── tasks.c
    ├── queue.c
    ├── list.c
    ├── timers.c
    ├── event_groups.c
    ├── stream_buffer.c
    └── croutine.c
```

### 3. FreeRTOSConfig.h 关键配置
```c
#define configCPU_CLOCK_HZ              SystemCoreClock     /* 168MHz */
#define configTICK_RATE_HZ              1000
#define configMAX_PRIORITIES            8
#define configMINIMAL_STACK_SIZE        128
#define configTOTAL_HEAP_SIZE           ((size_t)(32768))   /* 32KB */
#define configSUPPORT_STATIC_ALLOCATION 1
#define configSUPPORT_DYNAMIC_ALLOCATION 1
#define configCHECK_FOR_STACK_OVERFLOW  2
#define configUSE_MALLOC_FAILED_HOOK    1

/* 不定义 configSYSTICK_CLOCK_HZ，默认使用处理器时钟 */
/* Handler 映射 */
#define xPortPendSVHandler      PendSV_Handler
#define vPortSVCHandler         SVC_Handler
```

### 4. 官方 port.c 关键特性
- PendSV handler 检查 EXC_RETURN bit 4，按需保存/恢复 S16-S31
- SVC handler 从栈加载 `{r4-r11, r14}`（包含 EXC_RETURN）
- prvStartFirstTask 清除 CONTROL.FPCA 位
- prvEnableVFP 使能 CP10/CP11 协处理器
- xPortStartScheduler 设置 FPCCR ASPEN/LSPEN 位

## 三、应用层简化测试

### 最小化 main.c
```c
int main(void)
{
    NVIC_PriorityGroupConfig(NVIC_PriorityGroup_4);
    SysTick_Init(168);
    USART1_Init(115200);
    OLED_Init();    /* OLED 硬件测试：全屏白 */
    xTaskCreate(StartTask, "StartTask", 128, NULL, 4, &StartTaskHandle);
    vTaskStartScheduler();
    for(;;);
}
```

### 最小化 task_manager.c
- StartTask：创建 TestTask 后删除自身
- TestTask：每 2 秒打印 "[DBG] TestTask tick"
- 包含所有必要的 hook 函数（idle、stack overflow、malloc failed）
- 包含静态分配回调（Idle/Timer task）

## 四、OLED 驱动适配

从原 F103 项目复制 bsp_oled.c，修改以下内容：
1. `#include "stm32f10x.h"` → `#include "stm32f4xx.h"`
2. GPIO 引脚：PB8/PB9 → PB14/PB15
3. GPIO 初始化：F1 风格 → F4 风格
```c
// F103
RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE);
GPIO_InitStructure.GPIO_Mode = GPIO_Mode_Out_OD;

// F407
RCC_AHB1PeriphClockCmd(RCC_AHB1Periph_GPIOB, ENABLE);
GPIO_InitStructure.GPIO_Mode  = GPIO_Mode_OUT;
GPIO_InitStructure.GPIO_OType = GPIO_OType_OD;
GPIO_InitStructure.GPIO_PuPd  = GPIO_PuPd_UP;
```

## 七、关键经验总结

1. **F103→F407 必须更换 FreeRTOS 端口**：Cortex-M3 端口不支持 FPU，会导致 HardFault
2. **使用官方端口**：不要自己写 port.c，从 FreeRTOS-Kernel 仓库复制 RVDS/ARM_CM4F
3. **configSYSTICK_CLOCK_HZ 不要定义**：F407 默认使用处理器时钟（CLKSOURCE=1）
4. **先最小化测试**：先验证调度器能启动，再恢复完整应用
5. **软件 I2C 兼容性好**：F103 的软件 I2C 驱动只需改引脚和 GPIO 初始化即可在 F407 上使用

## 八、OTA 双分区固件更新系统实现

### 1. 新增 BSP 驱动
| 文件 | 功能 |
|------|------|
| `bsp_spi1.c/h` | SPI1 驱动（PA5/PA6/PA7/PC13） |
| `bsp_w25q128.c/h` | W25Q128 Flash 驱动（JEDEC ID: 0xEF4018） |
| `bsp_i2c_hw.c/h` | 硬件 I2C1 驱动（PB6/PB7） |
| `bsp_at24c02.c/h` | AT24C02 EEPROM + BootFlags 结构体 |
| `bsp_usart3.c/h` | USART3 驱动（PB10/PB11，ESP8266 通信） |
| `esp8266.c/h` | ESP8266 AT 指令框架 |
| `ota_manager.c/h` | OTA 下载、校验、标志管理 |

### 2. Bootloader 工程
- 裸机工程（无 FreeRTOS），64KB，运行在 0x08000000
- 全部使用 USART1 寄存器直接操作输出（不用 printf）
- SysTick 使用纯轮询模式（不使能中断）
- 检测 OTA 标志 → CRC32 校验 → 写入目标分区 → 跳转

### 3. App 工程修改
- 散列文件改为 0x08010000，192KB
- VECT_TAB_OFFSET = 0x10000
- 新增 test_ota.c 测试模块（6 个测试函数）
- StartTask 堆栈从 80 增大到 512 words
