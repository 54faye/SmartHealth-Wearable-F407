# 系统技术架构设计文档 (System Architecture)

## 1. 硬件架构
本项目基于 STM32F407VET6 开发板构建，外设资源分配如下：

*   **主控芯片**: STM32F407VET6 (ARM Cortex-M4, 168MHz, 512KB Flash, 192KB SRAM + 4KB 备份 SRAM)
*   **显示模块**: 0.96寸 OLED (SSD1306 控制器, 软件I2C接口, SCL=PB14, SDA=PB15)
*   **传感器**: MPU6050 六轴传感器 (3轴加速度 + 3轴陀螺仪, 软件I2C接口, SCL=PB12, SDA=PB13)
*   **人机交互**: 独立按键 x 3 (PA0/PA1/PA4, GPIO 输入)
*   **调试接口**: SWD (Serial Wire Debug) + USART1 (PA9/PA10, 115200bps)
*   **实时时钟**: RTC + 32.768kHz LSE晶振 (PC14/PC15)
*   **系统时钟**: HSE 25MHz → PLL → 168MHz
*   **OTA 存储**: W25Q128 SPI Flash (PA5/PA6/PA7/PC13, 16MB, 固件暂存)
*   **WiFi 模块**: ESP8266 (USART3: PB10/PB11, 115200bps, AT指令)
*   **启动标志**: AT24C02 EEPROM (硬件I2C1: PB6/PB7, BootFlags存储)

## 2. 软件架构 (Layered Architecture)
项目采用严格的**分层解耦**设计，自下而上分为三层，层与层之间通过标准 API 通讯。

### 2.1 驱动层 (Drivers Layer)
位于 `Drivers/` 目录，负责直接操作硬件寄存器。
*   **CMSIS & StdPeriph**: ARM 与 ST 官方提供的标准库（STM32F4xx StdPeriph）。
*   **BSP (Board Support Package)**: 板级支持包，封装具体硬件细节。
    *   `bsp_oled.c`: 提供绘图点、线、图的底层接口（软件I2C）。
    *   `bsp_mpu6050.c`: 封装 I2C 通信协议，提供原始数据读取接口（软件I2C）。
    *   `bsp_key.c`: 提供按键状态扫描接口（TIM2定时器轮询，10ms周期）。
    *   `bsp_led.c`: LED 控制（PC5，低电平点亮）。
    *   `bsp_usart.c`: USART1 串口驱动（115200bps，printf重定向）。
    *   `bsp_delay.c`: 系统延时（基于SysTick，适配168MHz）。
    *   `bsp_timer.c`: TIM2 定时器（10ms中断，驱动按键轮询和秒表）。
    *   `bsp_rtc.c`: RTC 实时时钟（日历型，备份寄存器存储时间戳基准）。
    *   `bsp_adc.c`: ADC 电压采集（PA0与KEY1冲突，当前不可用）。
    *   `bsp_spi1.c`: SPI1 驱动（PA5/PA6/PA7，用于 W25Q128 通信）。
    *   `bsp_w25q128.c`: W25Q128 SPI Flash 驱动（读/写/擦除，JEDEC ID 0xEF4018）。
    *   `bsp_i2c_hw.c`: 硬件 I2C1 驱动（PB6/PB7，用于 AT24C02 通信）。
    *   `bsp_at24c02.c`: AT24C02 EEPROM 驱动（BootFlags 读写，32字节结构体）。
    *   `bsp_usart3.c`: USART3 驱动（PB10/PB11，用于 ESP8266 通信）。
    *   `esp8266.c`: ESP8266 AT 指令框架（WiFi 连接、TCP 通信、HTTP 响应解析）。
    *   `ota_manager.c`: OTA 管理器（固件下载、CRC32 校验、Boot 标志设置）。
    *   **设计优势**: 上层业务逻辑无需关心底层是硬件 I2C 还是模拟 I2C，便于硬件移植。

### 2.2 中间件层 (Middlewares Layer)
位于 `Middlewares/` 目录，提供独立于硬件的通用服务。
*   **FreeRTOS**: 实时操作系统内核（ARM_CM4F移植层，堆大小32KB），负责任务调度、时间管理、任务间同步与通信。
*   **Algorithm**: 纯算法模块。
    *   `kalman.c`: 标准卡尔曼滤波算法实现。
    *   `step_algo.c`: 计步检测算法。
    *   `pedometer.c`: 计步器封装。

### 2.3 应用层 (App Layer)
位于 `App/` 目录，采用类 **MVC (Model-View-Controller)** 思想设计。
*   **Model (数据中心)**: `App/Model/`
    *   `DataCenter.c`: 系统的"单一数据源"，管理传感器数据、时间、系统设置等。使用 **Mutex (互斥量)** 保证多任务环境下的数据线程安全。
    *   `SetTime.c`: 时间设置逻辑。
*   **View (UI 视图)**: `App/UI/`
    *   `UI_Manager.c`: 页面状态机，管理页面切换逻辑。
    *   `Pages/`: 各个功能页面的绘制实现（`Page_Clock`, `Page_Menu`, `Page_Apps`）。
    *   `menu.c`: 菜单绘制与动画。
    *   `dino.c`: 恐龙游戏逻辑。
    *   采用**显存缓冲 (FrameBuffer)** 机制，一次性刷新，避免屏幕闪烁。
*   **Controller (任务逻辑)**: `App/Tasks/`
    *   `task_display.c`: OLED 显示任务。
    *   `task_sensor.c`: 传感器数据采集任务。
    *   `task_input.c`: 按键输入任务。
    *   `task_manager.c`: 系统管理任务。

## 3. 双工程架构 (Bootloader + App)

系统采用 **Bootloader + App** 双工程架构，支持 OTA 双分区固件更新。

### 3.1 Bootloader 工程 (`Bootloader/`)
- **职责**：检测 OTA 标志、固件拷贝、跳转到 App
- **运行地址**：0x08000000（内部 Flash Sector 0-3，64KB）
- **特点**：裸机运行（无 FreeRTOS），直接操作 USART1 寄存器输出调试信息
- **依赖**：bsp_spi1、bsp_w25q128、bsp_i2c_hw、bsp_at24c02、bsp_delay、bsp_usart

### 3.2 App 工程 (`Project/`)
- **职责**：手表主应用（UI、传感器、游戏等）
- **运行地址**：0x08010000（内部 Flash Sector 4-5，192KB）
- **特点**：FreeRTOS 多任务调度，支持 OTA 固件下载

### 3.3 分区表
```
0x08000000 ┌────────────────┐ 64KB
           │  Bootloader    │
0x08010000 ├────────────────┤ 192KB
           │  App_A         │
0x08040000 ├────────────────┤ 256KB
           │  App_B         │
0x08080000 └────────────────┘
```

### 3.4 OTA 流程
1. App 通过 ESP8266 下载固件到 W25Q128（HTTP GET）
2. App 设置 BootFlags（ota_pending=1, ota_target=非活动分区）
3. App 重启
4. Bootloader 检测到 ota_pending=1
5. 从 W25Q128 读取固件，CRC32 校验
6. 擦除目标分区，逐字写入内部 Flash
7. 更新 active_partition，清除 ota_pending
8. 跳转到新分区运行

## 4. FreeRTOS 任务划分
系统根据实时性要求，划分了以下核心任务：

| 任务名称 | 优先级 | 堆栈大小 | 调度频率 | 职责描述 |
| :--- | :--- | :--- | :--- | :--- |
| **Task_Display** | High (4) | 256 Words | 30Hz | 负责 OLED 显存刷新，高优先级保证 UI 流畅度，无撕裂。 |
| **Task_Sensor** | Medium (3) | 128 Words | 200Hz | 读取 MPU6050 数据，执行卡尔曼滤波解算，更新 DataCenter。 |
| **Task_Input** | Medium (3) | 128 Words | 50Hz | 扫描按键状态，执行消抖，向系统发送按键事件。 |
| **Task_Manager** | Low (2) | 128 Words | 10Hz | 系统状态监控，低功耗管理，OTA 测试入口。 |

## 5. 从 F103 到 F407 的迁移
本项目最初基于 STM32F103C8T6 开发，后迁移至 STM32F407VET6。迁移涉及以下关键变更：

| 项目 | F103 (旧) | F407 (新) |
|------|-----------|-----------|
| 内核 | Cortex-M3, 72MHz | Cortex-M4F, 168MHz |
| Flash/SRAM | 64KB / 20KB | 512KB / 192KB |
| GPIO时钟 | RCC_APB2Periph | RCC_AHB1Periph |
| GPIO配置 | 单结构体 | 模式/类型/上下拉分离 |
| RTC | 计数器型 | 日历型 |
| FreeRTOS堆 | 6KB | 32KB |
| StdPeriph库 | stm32f10x | stm32f4xx |

详细迁移记录见 `本次会话总结.md` 和 `技术困难攻克.md`。
